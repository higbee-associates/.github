# Security Policy

## Supported versions

Security updates are applied to actively maintained branches of Higbee Associates repositories. Check each repository’s README or release notes for the currently supported versions and environments.

## Reporting a vulnerability

**Do not** open a public GitHub issue for security vulnerabilities.

Please report security issues privately using one of these channels (prefer the first available):

1. **GitHub private vulnerability reporting** — if enabled on the repository, use **Security → Advisories → Report a vulnerability**.
2. **Email** — send details to the Higbee security contact used by your team (ask your lead if you do not have it).
3. **Internal tracker** — use the private security / incident process documented for your product team.

### What to include

- Affected repository, environment, and approximate version or commit
- Description of the issue and potential impact
- Steps to reproduce (proof-of-concept if available)
- Any suggested remediation

### What to expect

- Acknowledgement of your report as soon as practical
- An initial assessment of severity and scope
- Coordination on fix timing and disclosure (if any)

Please give us reasonable time to investigate and remediate before any public discussion.

## Safe harbor

We welcome good-faith security research. Avoid actions that degrade service availability, access data beyond what is needed to demonstrate the issue, or expose client or customer information.
