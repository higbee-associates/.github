# Support

This document describes how to get help with Higbee Associates projects. Individual repositories may override these defaults.

## How to get help

| Need | Where to go |
| --- | --- |
| Bug in a product repository | Open a **Bug report** issue using the template |
| New capability or improvement | Open a **Feature request** issue |
| How do I… / setup questions | Ask in the team channel for that product, or open a **Task** if work is needed |
| Security concern | Follow [`SECURITY.md`](./SECURITY.md) — do not post publicly |
| Access / permissions | Contact your team lead or organization admins |

## Before opening an issue

1. Check the repository `README.md` and `docs/`.
2. Search existing issues for duplicates.
3. Include environment details (branch, environment name, relevant IDs) without pasting secrets.

## Response expectations

Support is provided by the maintainers of each repository during normal business hours. Urgent production incidents should follow your team’s on-call / incident process rather than GitHub issues alone.
